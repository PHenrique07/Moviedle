1. Elementos de Texto e Estrutura

Entendendo o Versionamento

O Git é um sistema de controle de versão distribuído gratuito e de código aberto projetado para lidar com tudo, desde projetos pequenos até muito grandes com velocidade e eficiência. Ele é fácil de aprender e tem uma pegada minúscula com desempenho extremamente rápido.

"O Git superou outros sistemas SCM como Subversion, CVS, Perforce e ClearCase com recursos como ramificação (branching) local barata, áreas de teste (staging areas) convenientes e múltiplos fluxos de trabalho."

Principais Comandos do Git que você pode testar:

  • git add . - Adiciona os arquivos modificados ao stage.
  • git commit -m "mensagem" - Cria o commit com suas alterações.
  • git push origin main - Envia para o repositório remoto.
  • git status - Verifica o estado atual dos arquivos.

Passos para um bom commit:

  1. Entenda o que foi modificado.
  2. Separe alterações lógicas em commits diferentes.
  3. Escreva uma mensagem clara e concisa no imperativo.
